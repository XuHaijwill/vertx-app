# 2026-05-01 10:28 — 审计日志归档机制实现

## 目标

实现审计日志归档策略：
- `audit_logs` 保留最近 1 年
- `audit_logs_archive` 保存 1 年以上历史
- 大于 6 个月的日志自动迁移到 archive 表

## 实现内容

### 1. 数据库层 (V9__audit_logs_archive.sql)

**归档表**：
```sql
CREATE TABLE audit_logs_archive (LIKE audit_logs INCLUDING ALL);
ALTER TABLE audit_logs_archive ADD COLUMN archived_at TIMESTAMPTZ;
```

**归档函数**：
```sql
SELECT archive_audit_logs(6, 10000);  -- 迁移 > 6 个月的日志
SELECT purge_old_audit_logs(3);       -- 删除 > 3 年的归档
```

**统一视图**：
```sql
SELECT * FROM audit_logs_all;  -- 主表 + 归档表 UNION
```

### 2. Java 层

**AuditArchiveTask.java**：
- `archive(Vertx, JsonObject)` — 归档 > N 个月日志
- `purge(Vertx, JsonObject)` — 清理 > N 年归档
- `stats(Vertx, JsonObject)` — 归档统计

**AuditRepository.java 新增**：
- `getArchiveStats()` — 获取归档统计
- `archiveOldLogs(monthsOld, batchSize)` — 执行归档
- `purgeOldArchives(yearsOld)` — 清理历史

**AuditApi.java 新增端点**：
- `GET /api/audit-logs/stats/archive` — 归档统计
- `POST /api/audit-logs/archive` — 手动触发归档

### 3. 定时任务配置

## 使用方式

### 查询归档统计
```bash
GET /api/audit-logs/stats/archive
# 返回: {activeCount, archivedCount, oldestActive, oldestArchived, eligibleForArchive}
```

### 手动触发归档
```bash
POST /api/audit-logs/archive
{"monthsOld": 6, "batchSize": 10000}
# 返回: {archivedCount, monthsOld}
```

### 查询所有审计（含归档）
```sql
SELECT * FROM audit_logs_all WHERE entity_type = 'orders';
```
